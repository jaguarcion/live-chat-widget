# Техническое задание: Онлайн-чат для сайта (аналог Chatra)

## 0) Цель проекта

Разработать систему онлайн-чата для интеграции на любые сайты (в т.ч.
WordPress) по модели "как chatra.io":

-   Виджет на сайте: кнопка/бейдж "Напишите нам, мы онлайн!", окно чата,
    форма оффлайн-сообщения.
-   Кабинет оператора: список диалогов, чат, данные посетителя, заметки,
    распределение по операторам, статусы онлайн.
-   Серверная часть: realtime-сообщения, хранение истории, управление
    операторами и настройками.
-   Мультисайт: один аккаунт может иметь несколько сайтов (projects).

Готовый результат: продакшен‑вариант, пригодный к установке на сервер и
подключению на сайт одной строкой.

------------------------------------------------------------------------

# 1) Роли и права доступа

## Роли

1.  Owner --- полный доступ ко всем настройкам и проектам.
2.  Admin --- управление проектом и операторами.
3.  Operator --- работа с диалогами.
4.  Visitor --- посетитель сайта.

## Модель доступов

-   Все действия происходят в контексте Project.
-   Один оператор может быть в нескольких проектах.
-   Оператор видит только свои проекты.

------------------------------------------------------------------------

# 2) Основные сценарии

## Посетитель

-   Нажимает кнопку чата.
-   Пишет сообщение.
-   Если операторы онлайн --- получает ответ realtime.
-   Если оффлайн --- оставляет email + сообщение.
-   История сохраняется между перезагрузками.

## Оператор

-   Логинится в кабинет.
-   Видит список диалогов.
-   Отвечает пользователям.
-   Может назначить диалог оператору.
-   Видит данные посетителя.

## Администратор

-   Создает проекты.
-   Получает код вставки.
-   Настраивает внешний вид виджета.
-   Управляет операторами.

# 11) Acceptance Criteria

1.  Виджет подключается одной строкой.
2.  Посетитель может написать сообщение.
3.  Оператор получает сообщение realtime.
4.  История сохраняется.
5.  Оффлайн режим работает.
6.  Несколько операторов поддерживаются.
7.  Система разворачивается через Docker Compose.
